kesimpulan atau hasil pertemuan 2 yang saya ambil adalah:
1. Membuat repository dengan folder, awalnya add file kemudian pilih creat new file, setelah itu membuat folder dengan "nama folder"/, garis miring digunakan untuk membuat folder dalam github
2. import databse sql dan materi pembelajaran pada pertemuan pertama.
3. wordpress terdiri dari 2 sisi, yaitu admin dan klien area. jika menggunakan wp-admin maka kita dapat mengakses dari data pada pertemuan pertama, jika klien kita hanya dapat menampilkan layar depannya saja, tidak dapat mengubahnya.
4. setelah masuk dengan wp-admin, buka profile pengguna untuk megakses visual dari web kita, pada opsi personal jangan mengnonaktifkan penyunting visual agar yang muncul bukan kodingannya.
5. pintasan keyboard untuk komentar.
6. bilah perkakas ditampilkan jika memasukan dengan admin.
7. kemudian masuk kemateri nama pengguna, kemudian memasukan data data yang ada.
8. masuk ke pengguna, kemudian memasukan data data yang harus di isi.
9. peranan pada pengguna di bagikan beberapa, seperti
- pelanggan, bisa mengetahui aktivitas dari dashboard yang ada, karena akitivitas tersebut dapat masuk ke peranan pelanggan. kelebihannya dapat update profile dan password yang ada.
- kontributor, bisa melihat komen, menu peralatan, serta pos, karena peranana ini dapat melihat pos yang lain, dapat menambah pos kemudian di terbitkan. kontributor itu membutuhkan persetujuan dari editor dan admin, jika sudah kontributor tidak dapat mengedit posnya karena sudah di serahkan kepada editor dan admin.
- penulis, bisa membuat pos tanpa di tinjau oleh admin.
- editor, dapat disebut juga tangan kanan admin, karena bisa menambah, mengubah, dihapus posan.
10. pengaturan umum, mengisi data sesuai yang di perlukan.
11. jika keanggotaan diaktifkan maka pengguna dapat lebih dari 1.
12. pengaturan zona waktu indonesia "UTC+7", untuk tanggal dan waktu bebas, sesuai selera, kemudian di simpan.
13. kategori dapat sesuai dengan materi yang mau di tampilkan.
14. tampilan beranda anda, bebas memilih data yang disediakan.
15. jika kita memilih kutipan maka akan menjadi ringkas.
16. diskusi itu sama dengan komentar, pengaturan pada komentar dapat di tentukan sesuai kehendak admin.
17. pengaturan media sesuai juga dengan kehendak admin.
18. peringatan untuk website kita.

  1. di dalam htdocs, mesti ada folder si5a2122o, sehingga pada browser bisa diakses dengan http://localhost/si5a2122o
  2. jika object not found (404), maka cek dulu nama folder antara htdocs dengan yang diketik di localhost
  3. jika ada tulisan error establishing database connection, cek dulu apakah mysql sudah running pada xampp control panel
  4. jika sudah running, buka wp-config.php pada folder si5a2122o, dan cek db_name. db name ini pastikan ada di phpmyadmin
  5. jika pada phpmyadmin belum ada nama databasenya, maka import dulu file sql bahan pertemuan ke 2 (hasil pertemuan ke 1)
  6. wordpress memiliki 2 sisi, 1 tampilan backend (untuk admin/dashboard) dan 1 nya lagi tampilan front end (untuk pengunjung/client area)
  7. untuk backend, harus login dulu di alamat http://localhost/si5a2122o/wp-admin
